This package contains the \char`\"{}\+Direct\+X Tool Kit\char`\"{}, a collection of helper classes for writing Direct3D 11 C++ code for Universal Windows Platform (UWP) apps for Windows 11, Windows 10, Xbox One, and Win32 desktop applications for Windows 7 Service Pack 1 or later.

This code is designed to build with Visual Studio 2019 (16.\+9 or later), Visual Studio 2022, or clang for Windows v11 or later. Use of the Windows 10 May 2020 Update SDK (\href{https://walbourn.github.io/windows-10-may-2020-update-sdk/}{\texttt{ 19041}}) or later is required.

These components are designed to work without requiring any content from the legacy DirectX SDK. For details, see \href{https://aka.ms/dxsdk}{\texttt{ Where is the DirectX SDK?}}.\hypertarget{md__c___users__family_source_repos__liquid_engine__liquid_engine__graphics__direct_x_t_k__r_e_a_d_m_e_autotoc_md93}{}\doxysubsection{Directory Layout}\label{md__c___users__family_source_repos__liquid_engine__liquid_engine__graphics__direct_x_t_k__r_e_a_d_m_e_autotoc_md93}

\begin{DoxyItemize}
\item As of the March 2022 release, legacy Xbox One XDK support requires the XDK April 2018 release or later.
\item As of the October 2021 release, the DGSLEffect no longer directly supports skinning. Instead, make use of {\bfseries{Skinned\+DGSLEffect}} which is derived from DGSLEffect.
\item As of the August 2021 release, {\bfseries{\mbox{\hyperlink{namespace_make_sprite_font}{Make\+Sprite\+Font}}}} requires the .NET version 4.\+7.\+2 targeting pack to be installed to build.
\item Starting with the June 2020 release, this library makes use of \href{https://walbourn.github.io/modern-c++-bitmask-types/}{\texttt{ typed enum bitmask flags}} per the recommendation of the {\itshape C++ Standard} section {\itshape 17.\+5.\+2.\+1.\+3 Bitmask types}. This may have {\itshape breaking change} impacts to client code\+:
\begin{DoxyItemize}
\item You cannot pass the {\ttfamily 0} literal as your flags value. Instead you must make use of the appropriate default enum value\+: {\ttfamily Audio\+Engine\+\_\+\+Default}, {\ttfamily Sound\+Effect\+Instance\+\_\+\+Default}, {\ttfamily Model\+Loader\+\_\+\+Clockwise}, or {\ttfamily WIC\+\_\+\+LOADER\+\_\+\+DEFAULT}.
\item Use the enum type instead of {\ttfamily DWORD} if building up flags values locally with bitmask operations. For example, {\ttfamily WIC\+\_\+\+LOADER\+\_\+\+FLAGS flags = WIC\+\_\+\+LOADER\+\_\+\+DEFAULT; if (...) flags $\vert$= WIC\+\_\+\+LOADER\+\_\+\+FORCE\+\_\+\+SRGB;}
\end{DoxyItemize}
\item The UWP projects and the Win10 classic desktop project include configurations for the ARM64 platform. Building these requires installing the ARM64 toolset.
\item When using clang/\+LLVM for the ARM64 platform, the Windows 11 SDK (\href{https://walbourn.github.io/windows-sdk-for-windows-11/}{\texttt{ 22000}}) is required.
\item The {\ttfamily Compile\+Shaders.\+cmd} script must have Windows-\/style (CRLF) line-\/endings. If it is changed to Linux-\/style (LF) line-\/endings, it can fail to build all the required shaders.
\end{DoxyItemize}\hypertarget{md__c___users__family_source_repos__liquid_engine__liquid_engine__graphics__direct_x_t_k__r_e_a_d_m_e_autotoc_md97}{}\doxysubsection{Support}\label{md__c___users__family_source_repos__liquid_engine__liquid_engine__graphics__direct_x_t_k__r_e_a_d_m_e_autotoc_md97}
